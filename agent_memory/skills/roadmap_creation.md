# Skill: Roadmap Creation

## Description
Ability to create comprehensive implementation roadmaps from requirements documents, particularly for technical projects and exercises.

## Learned From
- User request: "In the folder casus_data is a pdf file with an excersise. For this excersise I want to create a roadmap (roadmap.md) to make steps of completing this excersise."
- User feedback: "Very nice thanks"

## Key Components
1. **Requirements Analysis**: Reading and understanding PDF requirements documents
2. **Structured Breakdown**: Organizing requirements into functional (P1-P8) and non-functional (S1-S4) categories
3. **Timeline Planning**: Creating day-by-day implementation plans
4. **Progress Tracking**: Incorporating checkboxes for task completion tracking
5. **Success Criteria**: Defining clear metrics for project completion

## Implementation Pattern
1. Analyze source material (PDF, documentation, requirements)
2. Extract key requirements and categorize them
3. Create structured overview section
4. Break down into actionable tasks with timeline
5. Include deliverables and success criteria
6. Format with markdown and checkboxes for clarity

## Example Output Structure
```
# Roadmap for [Project Name]

## Exercise Overview
[Brief description]

## Functional Requirements
- [Requirement 1]
- [Requirement 2]
- ...

## Non-functional Requirements
- [Requirement 1]
- [Requirement 2]
- ...

## Deliverables
1. [Deliverable 1]
2. [Deliverable 2]
...

## Implementation Roadmap
### Day 1: [Phase Name]
- [ ] Task 1
- [ ] Task 2
...

## Success Criteria
- [Criterion 1]
- [Criterion 2]
...
```

## User Preferences Observed
- Appreciates comprehensive, detailed roadmaps
- Values clear progress tracking (checkboxes)
- Prefers technical detail and structure
- Likes step-by-step breakdowns with clear milestones
